# Definition Of Done

A feature is done only when it is implemented, reviewed, and manually tested against its acceptance criteria.


## Required For Every Feature

- [ ] Feature goal is clear
- [ ] Business rules are documented or confirmed
- [ ] Acceptance criteria are written
- [ ] Manual QA checklist is prepared
- [ ] Code is implemented only for the current task
- [ ] No unrelated refactor was added
- [ ] User-facing text is clear
- [ ] Error handling is considered
- [ ] Security and authorization risks are considered
- [ ] Manual testing was performed
- [ ] Bugs found during testing are documented
- [ ] Documentation is updated if behavior changed

## Before Moving To The Next Feature

- [ ] Happy Path was tested
- [ ] Negative Testing was considered
- [ ] Edge Cases were considered
- [ ] UI/UX was checked
- [ ] Accessibility was checked at a basic level
- [ ] Regression risk was considered

## Later, When Automated Tests Are Added

- [ ] Unit tests are added when useful
- [ ] API tests are added for backend behavior
- [ ] E2E tests are added for critical user flows
- [ ] CI checks pass
